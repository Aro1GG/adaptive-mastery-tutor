# 计算机网络 — 传输层

## 传输层的角色

传输层提供端到端的逻辑通信。网络层负责主机到主机，而传输层负责进程到进程。
核心功能：复用（发送方收集多个进程数据）与分用（接收方分发到对应进程），
通过端口号（16 位整数，0-65535）标识进程。

UDP 是无连接的，尽最大努力交付，首部仅 8 字节。典型应用：DNS（端口 53）、
流媒体、广播。

## 可靠传输原理

TCP 提供可靠传输。核心机制：
- 停止等待协议：发一个包，等 ACK，收到才发下一个。简单但利用率低。
- 流水线传输：连续发送多个包再等确认。分为：
  - GBN（Go-Back-N）：一个丢失，重传从那之后的所有包
  - 选择重传（Selective Repeat）：只重传丢失的包

## TCP 连接管理

### 三次握手

1. 客户端 → 服务器：SYN=1，seq=x（客户端选择初始序列号）
2. 服务器 → 客户端：SYN=1，ACK=1，seq=y，ack=x+1
3. 客户端 → 服务器：ACK=1，seq=x+1，ack=y+1

**为什么是三次不是两次？** 防止旧连接请求在网络中滞留导致服务端错误开启连接。

**SYN Flood 攻击：** 攻击者发送大量 SYN 但不完成握手，耗尽服务器半连接队列。
防御：SYN Cookie（不分配资源直到验证客户端真实性）。

### 四次挥手

1. 客户端 → 服务器：FIN=1（我没有数据了，但还能接收）
2. 服务器 → 客户端：ACK（我知道了）
3. 服务器 → 客户端：FIN=1（我也没有数据了）
4. 客户端 → 服务器：ACK → 进入 TIME_WAIT，等待 2MSL

**TIME_WAIT 为什么等 2MSL？** 确保最后一个 ACK 能重传（MSL=报文最大生存时间），
也确保本连接的所有报文从网络中消失。

## TCP 拥塞控制

拥塞控制 ≠ 流量控制：流量控制是接收方能力限制（rwnd），拥塞控制是网络能力限制（cwnd）。

四个算法：
1. **慢启动：** cwnd=1，每 RTT 翻倍，直到 ssthresh
2. **拥塞避免：** 超过 ssthresh 后线性增长（每 RTT +1 MSS）
3. **快重传：** 收到 3 个重复 ACK 立即重传（不等超时）
4. **快恢复：** 快重传后不降到 1，而是降到 ssthresh 的一半，进入拥塞避免

## 常见问题与诊断

- **TIME_WAIT 过多：** 高并发短连接服务常见，调整 net.ipv4.tcp_tw_reuse
- **连接被拒绝 vs 超时：** 拒绝 = RST 包（端口未监听），超时 = 包被丢弃/防火墙
- **端口耗尽：** 客户端端口范围有限（默认 32768-60999），高并发需调整或使用长连接
